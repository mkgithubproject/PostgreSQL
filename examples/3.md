Yes, you **can write this logic using pure JavaScript with TypeORM**, but what you mean by тАЬpure JSтАЭ usually refers to **not using QueryBuilder and instead using repository methods + JS calculations**.

However, in your case the query contains:

* `COUNT`
* `SUM`
* `subquery`
* `GROUP calculations`
* `ROUND`
* `COALESCE`

These are **SQL aggregation operations**, and **TypeORM repository methods (`find`, `findOne`) cannot efficiently perform these**. If you fetch all rows and calculate in JS, it becomes **slow and memory-heavy**.

So you have **3 possible approaches in TypeORM**:

---

# 1я╕ПтГг Your Current Approach (Best for this case)

Using **TypeORM QueryBuilder**.

Advantages:

* Runs aggregation in **PostgreSQL**
* **Fast**
* Only **1 query**
* Handles `SUM`, `COUNT`, `ROUND`, `COALESCE`

Your current code is actually **the correct industry approach**.

Example simplified:

```ts
const summary = await this.orderRepository
  .createQueryBuilder("orders")
  .select("COUNT(orders.order_id)", "numberOfOrders")
  .addSelect("SUM(orders.total_order_amount)", "orderSalesAmount")
  .where("orders.seller_code = :storeCode", { storeCode })
  .getRawOne();
```

# 3я╕ПтГг Raw SQL Query (Also Common)

Another approach is using **raw SQL**.

```ts
const summary = await this.orderRepository.query(`
SELECT 
COUNT(order_id) as "numberOfOrders",
COALESCE(SUM(total_order_amount),0) as "orderSalesAmount"
FROM ec_orders
WHERE seller_code = $1
AND shipment_completion_date >= $2
AND shipment_completion_date < $3
`, [storeCode, start, end]);
```

Advantages:

* Fast
* Full SQL control

Disadvantage:

* Less TypeORM abstraction
